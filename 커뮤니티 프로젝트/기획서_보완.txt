■■ 2. 데이터 출처 (★가장 먼저 결정) ■■

게시물(소설/잡지)의 제목·이미지·저자·가격 정보를
"누가, 어떻게" 채울지 결정해야 합니다.

  A. 관리자가 직접 등록  → 관리자 페이지 필요
  B. 사용자가 등록       → 누구나 게시물 작성 가능
  C. 외부 도서 API 연동  → 알라딘/네이버 책 API 자동 수집
  💡 학습/포트폴리오 목적이면 B 또는 C 추천

✍️ [작성필요] 우리가 선택할 방식 : B, C
    (A / B / C 중 택1, 또는 조합)

✍️ [작성필요] (C를 골랐다면) 사용할 API 이름 : 네이버 책 API


■■ 3. 회원 / 로그인 정책 ■■

✅ 회원가입 : 이메일, 비밀번호, 비밀번호 확인, 닉네임
✅ 이메일 중복 검사 기능
✅ 로그인 : 이메일 + 비밀번호

▷ 비로그인 사용자가 할 수 있는 것 (열람 vs 참여)
  💡 제안: 게시물/댓글 "보기"는 누구나 가능,
           "댓글·평점·좋아요 작성"은 로그인 필요

✍️ [작성필요] 비로그인 사용자 권한 정하기
    - 게시물 목록/상세 보기 : (허용)
    - 댓글 작성             : (불가)
    - 평점 주기             : (불가)

✍️ [작성필요] 헤더(상단 바) 표시 내용
    - 로그인 전 : [로그인] [회원가입] 버튼
    - 로그인 후 : 닉네임, [마이페이지], [로그아웃] 등
    → 원하는 구성: ____________________________


■■ 4. 평점 시스템 규칙 (★모호했던 부분) ■■

✅ 소설/잡지별 평점 표시
✅ 댓글마다 작성자가 평점 부여

▷ 규칙을 명확히 정해야 합니다.
    - 척도    : 별점 5점 만점 1점 단위
    - 대표 평점 = 사용자들이 준 평점의 "평균"
    - 1인 1평점 (재평가 시 기존 점수 수정)
    - 댓글과 평점을 함께 남기는 방식
