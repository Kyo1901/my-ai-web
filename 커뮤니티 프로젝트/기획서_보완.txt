■■ 1. 프로젝트 개요 ■■

✅ 사이트 이름 : Novel_Community
✅ 주제       : 소설/잡지 정보 공유 및 평점 커뮤니티
✅ 색상 테마   : 파스텔톤의 푸른색
✅ 분위기      : 친근하고 재밌는 분위기
✅ 참고 사이트 : 교보문고, 네이버 카페

✍️ [작성필요] 한 줄 소개 (사이트 대표 문구/슬로건)
    예) "함께 읽고, 함께 평가하는 소설·잡지 놀이터"
    → 여기에 작성: "함께 읽고, 함께 평가하는 소설·잡지 놀이터"


■■ 2. 데이터 출처 (★가장 먼저 결정) ■■

게시물(소설/잡지)의 제목·이미지·저자·가격 정보를
"누가, 어떻게" 채울지 결정해야 합니다.

  A. 관리자가 직접 등록  → 관리자 페이지 필요
  B. 사용자가 등록       → 누구나 게시물 작성 가능
  C. 외부 도서 API 연동  → 알라딘/네이버 책 API 자동 수집
  💡 학습/포트폴리오 목적이면 B 또는 C 추천

✅ [확정] 우리가 선택할 방식 : A + C 조합 (신청 게시판 방식)
    - 사용자가 직접 게시물을 작성하지는 않음 (B 제외)
    - 대신 사용자가 "신청 게시판"에서 원하는 소설/잡지를 신청
    - 관리자가 신청 내역을 확인 → 네이버 책 API로 정보 조회 → 게시물로 등록
    → 상세 흐름은 아래 5-8. 신청 게시판 참고

✅ [확정] 사용할 API 이름 : 네이버 책 API
    (API 키 발급 및 연동 코드 작성은 기획 완료 후 실제 개발 단계에서 진행)

✅ [확정] 관리자 계정 : 일반 회원 계정과 별도로 전용 관리자 계정을 준비
    - 회원가입으로 생성되는 일반 계정과 분리하여 별도 발급/관리
    - 신청 게시판 승인·반려, 게시물 등록 권한은 이 관리자 계정에서만 동작

■■ 3. 회원 / 로그인 정책 ■■

✅ 회원가입 : 이메일, 비밀번호, 비밀번호 확인, 닉네임
✅ 이메일 중복 검사 기능
✅ 로그인 : 이메일 + 비밀번호

▷ 비로그인 사용자가 할 수 있는 것 (열람 vs 참여)
  💡 제안: 게시물/댓글 "보기"는 누구나 가능,
           "댓글·평점·좋아요 작성"은 로그인 필요

✍️ [작성필요] 비로그인 사용자 권한 정하기
    - 게시물 목록/상세 보기 : (허용)
    - 댓글 작성             : (불가)
    - 평점 주기             : (불가)

✍️ [작성필요] 헤더(상단 바) 표시 내용
    - 로그인 전 : [로그인] [회원가입] 버튼
    - 로그인 후 : 닉네임, [마이페이지], [로그아웃] 등
    → 원하는 구성: 로그인 전 = 우측에 [로그인] [회원가입] 버튼 두 개 노출
                  로그인 후 = 우측에 "○○○님" 닉네임 표시 → 클릭 시 [마이페이지] [로그아웃] 드롭다운 메뉴


■■ 4. 평점 시스템 규칙 (★모호했던 부분) ■■

✅ 소설/잡지별 평점 표시
✅ 댓글마다 작성자가 평점 부여

▷ 규칙을 명확히 정해야 합니다.
    - 척도    : 별점 5점 만점 1점 단위
    - 대표 평점 = 사용자들이 준 평점의 "평균"
    - 1인 1평점 (재평가 시 기존 점수 수정)
    - 댓글과 평점을 함께 남기는 방식



■■ 5. 페이지 구성 ■■

── 5-1. 메인 페이지 ──
✅ 신간 소설/잡지 표시
✅ 인기 소설/잡지 표시
✅ 인기 장르 5개, 장르별 소설 5~10개
✅ 왼쪽 사이드바 : 장르 / 잡지 종류 카테고리
✅ 검색창
  ▷ "인기"의 기준이 필요 (아래에서 정렬 기준으로 결정)

── 5-2. 로그인 페이지 ──
✅ 이메일, 비밀번호 입력 + 로그인 버튼
✍️ [작성필요] "회원가입으로 이동" 링크 위치/문구
                - 로그인 버튼 바로 아래 중앙에 "아직 계정이 없으신가요? 회원가입" 텍스트 링크로 배치
✍️ [작성필요] 비밀번호 찾기 기능 넣을지 : 예

── 5-3. 회원가입 페이지 ──
✅ 이메일, 이메일 중복검사, 비밀번호, 비밀번호 확인, 닉네임
✍️ [작성필요] 닉네임 중복검사도 넣을지 : 예
    (닉네임은 댓글·평점에 항상 노출되므로 이메일과 동일하게 실시간 중복 확인 버튼 제공)

── 5-4. 게시물 목록 페이지 ──
✅ 게시물 카드 정보:
     제목 / 이미지 / 저자·출판사 / 가격 / 평점 / 댓글 수
✅ 왼쪽 사이드바 : 장르별 카테고리
▷ [추가] 정렬 옵션 필요
  💡 제안: 인기순 / 신간순 / 평점순 / 댓글많은순
▷ [추가] 페이지네이션(페이지 넘김) 또는 무한스크롤

✍️ [작성필요] 정렬 옵션 목록 : 인기순(기본값) / 신간순 / 평점순 / 댓글많은순
✍️ [작성필요] 목록 표시 방식 : 페이지네이션 (페이지당 12~20개, 하단에 페이지 번호)
    💡 무한스크롤보다 구현이 간단해 1차 개발 추천, 필요하면 2차에서 무한스크롤로 교체 가능

── 5-5. 게시물 상세 페이지 ──
✅ 제목 / 이미지 / 저자·출판사 / 가격 / 평점 / 간단한 줄거리
✅ 댓글 영역:
     작성자 / 내용 / 작성시간 / 작성자가 부여한 평점

── 5-6. 검색 결과 페이지 [추가] ──
▷ 검색 기능은 있으나 결과 페이지가 없었음
✍️ [작성필요] 검색 대상 : 제목 + 저자 + 출판사 (통합 검색)
✍️ [작성필요] 검색 결과 없을 때 화면 문구 : "'○○○'에 대한 검색 결과가 없어요. 다른 검색어로 다시 시도해보세요 🔍"

── 5-7. 마이페이지 [추가] ──
  💡 제안 항목: 내가 쓴 댓글, 내가 준 평점, 찜한 목록, 닉네임 변경
✅ [확정] 마이페이지 : 3차 항목이므로 이번 개발 범위(2차까지)에서는 제외, 추후 별도 개발 예정
    (내 신청 현황은 5-8 신청 게시판에서 "내 신청만 보기"로 임시 대체 가능)

── 5-8. 신청 게시판 [추가] ──
  (2번 데이터 출처 방식이 "신청 게시판" 방식으로 정해지며 새로 필요해진 페이지)
✅ 사용자가 등록을 원하는 소설/잡지를 신청 : 도서명, 신청 사유(선택) 입력
✅ 신청 목록 표시 : 신청자 닉네임 / 신청 도서명 / 신청일 / 처리 상태(대기중 · 승인됨 · 반려됨)
✅ 상태 변경은 관리자 계정만 가능
    - 승인 시 : 관리자가 네이버 책 API로 해당 도서 조회 후 정보 확인하여 게시물로 등록
    - 반려 시 : 반려 사유 입력 (예: 중복 신청, 정보 부족 등)
  💡 로그인 사용자만 신청 가능 (비로그인은 3번 정책에 따라 불가)


■■ 6. 댓글 기능 ■■

✅ 대댓글 (댓글의 답글)
✅ 댓글 좋아요
✅ 베스트 댓글 상단 고정
▷ [추가] 본인 댓글 수정 / 삭제 기능
✍️ [작성필요] 베스트 댓글 기준 : 좋아요 5개 이상 받은 댓글 중 좋아요 최다 상위 3개까지 상단 고정


■■ 7. 추가 기능 아이디어 (선택) ■■
  💡 찜(북마크)   : 관심 소설 저장 → 마이페이지 확인
  💡 읽음 상태     : 읽고싶어요 / 읽는 중 / 읽음
  💡 자유 태그     : "힐링", "반전", "완결" 등

✅ [확정] 추가 기능 : 찜(북마크), 읽음 상태 표시, 자유 태그 모두 3차 항목 → 이번 개발 범위(2차까지)에서는 제외, 추후 개발 예정


■■ 8. 개발 우선순위 (범위 조절) ■■

기획 범위가 넓으므로 단계를 나눠 개발 추천.

  ▷ 1차 (핵심) :
      회원가입/로그인 → 게시물 목록 → 상세 → 댓글 → 평점
  ▷ 2차 (확장) :
      검색, 카테고리 필터, 대댓글, 좋아요, 베스트 댓글, 정렬
  ▷ 3차 (여유) :
      찜, 태그, 마이페이지, 읽음 상태

✅ [확정] 이번에 개발할 범위 : 2차까지
    - 1차(핵심) + 2차(검색/필터/정렬/대댓글/좋아요/베스트댓글) 전체 개발
    - 3차(찜, 태그, 마이페이지, 읽음 상태)는 이번 개발에서 제외 → 추후 별도 개발 예정
