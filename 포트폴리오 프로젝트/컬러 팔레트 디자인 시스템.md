  --color-primary-dark: #00CC82;   /* 어두운 그린 (눌림·활성) */

  /* Secondary Colors */

  /* Text Colors */
  --color-text-primary: #FFFFFF;   /* 제목·주요 텍스트 */
  --color-text-secondary: #B8BCC2; /* 본문·채널명 */

## 컬러 사용 가이드

각 색상을 언제, 어떻게 사용할지에 대한 구체적인 가이드라인입니다.

### 🟢 Primary (네온 그린 `#00FFA3`)
- **사용처**: 핵심 CTA 버튼, 활성 링크, 로고, 팔로우/선택된 상태, 강조하고 싶은 아이콘
- **주의**: 전체 화면의 **3~5% 이내**로만 사용. 어두운 배경 위에서 매우 강렬하게 인지되므로 남용하면 시각적 피로를 유발합니다. "여기를 클릭하세요"라는 신호로만 아껴 쓰세요.
- **금지**: 넓은 면적의 배경색으로 사용하지 말 것. 긴 본문 텍스트 색상으로 사용하지 말 것(가독성 저하).

### 🔴 Accent (LIVE 레드 `#FF1F3D`)
- **사용처**: 실시간/긴급 상태 배지, 알림 뱃지, 삭제·경고 등 주의가 필요한 액션
- **주의**: 그린과 동시에 큰 면적으로 쓰지 말 것(색상 충돌). 오직 "지금 일어나는 일"이라는 즉시성 신호로만 사용.

### ⬛ Background (다크 계열)
- `--color-bg-primary (#0E0E11)`: 페이지 전체 바탕
- `--color-bg-secondary (#18191D)`: 카드, 패널, 모달 등 콘텐츠를 담는 표면. 배경보다 살짝 밝게 하여 **깊이(elevation)** 표현
- 원칙: 요소가 위로 떠오를수록(카드 → 모달 → 팝오버) 배경을 조금씩 밝게 조정

### ⬜ Text (명도 위계)
- `--color-text-primary`: 제목·중요 정보 (최고 명도)
- `--color-text-secondary`: 일반 본문·설명
- `--color-text-muted`: 시간·조회수 등 보조 정보 (최저 명도)
- 원칙: **밝기 차이만으로 정보의 우선순위**를 표현. 색상을 늘리지 않고 위계를 만드는 다크 UI의 핵심 기법

### 🎯 Interactive (버튼·링크)
- 기본 → 호버 → 활성(눌림) 순으로 명도 변화를 줘서 상호작용을 피드백
- 버튼: `#00FFA3` → 호버 `#00CC82`(어두워짐)
- 링크: `#00FFA3` → 호버 `#5CFFC4`(밝아짐)

## 반응형 고려사항

다크모드나 다양한 화면에서의 색상 적용 방법입니다.

### 🌗 라이트/다크 모드 대응
이 팔레트는 **다크 모드가 기본**입니다. 라이트 모드도 지원하려면 `prefers-color-scheme`로 변수를 재정의하세요.

```css
/* 기본: 다크 모드 */
:root {
  --color-bg-primary: #0E0E11;
  --color-bg-secondary: #18191D;
  --color-text-primary: #FFFFFF;
  --color-text-secondary: #B8BCC2;
}

/* 사용자가 라이트 모드를 선호할 때 */
@media (prefers-color-scheme: light) {
  :root {
    --color-bg-primary: #FFFFFF;
    --color-bg-secondary: #F4F5F7;
    --color-text-primary: #141519;
    --color-text-secondary: #4A4E54;
    /* 그린은 라이트 배경에서 눈부시므로 약간 어둡게 */
    --color-primary: #00CC82;
    --color-link: #00A86B;
  }
}
```

### 📱 화면 크기별 고려
- **모바일**: OLED 화면에서 순수 검정(`#000`)은 스크롤 시 잔상이 생길 수 있어, `#0E0E11`처럼 아주 살짝 밝은 값을 사용하는 것이 안전
- **고휘도/야외**: 네온 그린은 밝은 환경에서 대비가 약해질 수 있으므로, 중요한 CTA는 색상만이 아니라 **배경 채움 + 텍스트 대비**로 이중 강조
- **저사양/저색역 디스플레이**: 핵심 정보는 색상에만 의존하지 말고 아이콘·텍스트 레이블을 함께 제공

### ♿ 접근성 (명암비)
- 본문 텍스트는 배경과 **최소 4.5:1** 명암비 확보 (`#B8BCC2` on `#0E0E11` ✅ 통과)
- 네온 그린 텍스트를 어두운 배경에 쓸 때는 큰 글자(18px 이상)에만 사용 권장
- 색맹 사용자를 위해 그린(정상)/레드(경고)를 상태 표시에 쓸 때는 **아이콘·모양**을 반드시 병행
