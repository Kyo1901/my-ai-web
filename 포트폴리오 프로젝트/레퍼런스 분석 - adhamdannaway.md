# 레퍼런스 포트폴리오 분석 — adhamdannaway.com

분석 대상: https://www.adhamdannaway.com/
분석 방법: 실제 화면 스크린샷(데스크톱/모바일) + 홈/About/Contact 페이지 콘텐츠 확인

## 1. 첫인상 분석

- **3초 인상**: "이 사람은 디자인도 하고 코딩도 한다"는 정체성이 즉각 전달됨. 반쪽은 팝아트 일러스트(디자이너), 반쪽은 실사 사진 위에 코드 텍스처(코더)로 얼굴을 분할한 히어로 이미지가 핵심.
- **가장 먼저 주목되는 요소**: 좌우로 "designer" / "&lt;coder&gt;" 대형 타이포와 그 사이의 반반 초상화. 텍스트보다 이미지 메타포가 시선을 먼저 붙잡는 구조.
- **톤앤매너**: 미니멀하지만 장난기 있는(playful) 톤. 블랙&화이트 기본 톤 위에 파스텔 브러시 컬러로 포인트를 줘서 "전문적이되 딱딱하지 않은" 인상.

## 2. 디자인 요소 분석

- **색상 팔레트**: 블랙(헤더) + 화이트(배경) 베이스에 코랄핑크·민트·머스터드 옐로·레드 브러시 스트로크를 악센트로 사용. 무채색 기본기 위에 채도 높은 포인트 컬러 소수 사용.
- **타이포그래피**: 헤드라인은 굵은 지오메트릭 산세리프. `<coder>`처럼 태그 괄호를 그대로 텍스트에 써서 "개발자다움"을 타이포로 표현.
- **레이아웃/여백**: 히어로 섹션에 여백을 넉넉히 두고 좌(디자이너 카피)-중(이미지)-우(코더 카피) 3분할 대칭 구조. 프로젝트 카드도 카드 사이 여백이 넓음.
- **시각적 계층구조**: 히어로(정체성) → "Some of my latest work"(대표작 3개) → footer 순으로 단순 명확.

## 3. UX 분석

- **네비게이션**: about / learn / portfolio / blog / contact 5개 + 소셜 아이콘. 모바일에서는 햄버거 메뉴로 축소.
- **정보 구조**: 홈은 "정체성 요약 + 대표작 3개"만 보여주고, About/Portfolio/Blog로 깊은 콘텐츠를 분산시키는 다중 페이지 구조. 홈을 가볍게 유지하는 전략.
- **인터랙션/애니메이션**: 프로젝트 섹션이 스크롤 진입 시 나타나는 페이드인 애니메이션으로 추정(스크린샷 캡처 시 실제 스크롤 없이는 해당 섹션이 빈 화면으로 잡힘). 연출은 좋지만 JS/스크롤 이벤트에 의존하는 콘텐츠는 접근성·SEO 크롤러 관점에서 약점이 될 수 있음.
- **모바일 반응형**: 히어로가 텍스트-이미지-텍스트 3분할에서 상하 스택으로 자연스럽게 재배치, 프로젝트 카드도 1열로 정리되어 반응형 대응 수준 준수.

## 4. 콘텐츠 전략 분석

- **자기소개 방식**: "Product designer" / "Front end developer" 두 정체성을 대등하게 병치. About 페이지에서는 "Yoda is my mentor" 같은 위트, 요리·운동·정원 가꾸기 같은 사생활 요소로 인간미를 더함.
- **프로젝트 소개**: 홈에서는 3개만 큐레이션(디자인 시스템, 책, 디자인 시스템 프로젝트)해 얕고 넓게 보여주고, 카테고리 태그로 유형을 즉시 파악하게 함.
- **CTA/연락처 접근성**: 홈에는 명시적 버튼형 CTA가 거의 없고, Contact 페이지는 소셜 링크 + 짧은 폼(이름/이메일/메시지)으로 진입장벽을 낮춤. 다만 제출 버튼이 "Δ" 기호만 있어 스크린리더 사용자에게는 불친절.
- **스토리텔링 구조**: 단순 이력서 나열이 아니라 "책 저술 → 디자인 시스템 → 이야기(My story)"처럼 콘텐츠를 자산화해 포트폴리오를 "일감을 위한 페이지"가 아니라 "개인 브랜드 허브"로 확장.

## 5. 장점과 개선점

### 강점 3가지
1. 반반 초상화라는 단일 비주얼 메타포로 "디자이너+코더" 정체성을 3초 안에 각인 — 카피 없이도 기억에 남음.
2. 책·디자인 시스템 등 포트폴리오 이상의 "사고 리더십(thought leadership)" 콘텐츠로 차별화.
3. 홈페이지를 가볍게 유지하고 깊은 내용은 하위 페이지로 위임하는 정보 구조 — 첫 방문자 피로도가 낮음.

### 아쉬운 점
- 스크롤 트리거 애니메이션에만 의존하는 섹션은 초기 렌더링/크롤링 관점에서 리스크.
- 홈에 명확한 행동 유도 버튼(예: "View Portfolio", "Hire Me")이 부족해 채용 담당자가 다음에 뭘 눌러야 할지 애매할 수 있음.
- Contact 폼 제출 버튼 라벨 접근성 부족.

### 내 포트폴리오에 적용할 아이디어
- 텍스트 위주 히어로 대신, "커뮤니티/SNS/포트폴리오를 만든 사람"이라는 정체성을 상징하는 단일 비주얼 모티프를 추가하면 기억에 남는 인상을 줄 수 있음.
- 프로젝트 카드에 카테고리 태그(예: "SNS", "커뮤니티")를 붙이면 방문자가 유형을 빠르게 스캔 가능.
- 방명록으로 이미 갖춘 "인간미" 요소처럼, About 페이지에도 위트 있는 한 줄(취미, 좌우명 등)을 넣으면 비슷한 효과.

### 차별화할 수 있는 방향성
- Adham은 "책/디자인 시스템"으로 권위를 세웠다면, 우리는 "Supabase 연동 실전 프로젝트(커뮤니티, SNS)"처럼 실제로 돌아가는 서비스를 Live Demo로 바로 체험시키는 것 자체가 차별점이 될 수 있음 — Projects 탭의 Live Demo 버튼을 더 강조하는 방향이 유효.

## 6. 타겟 분석

- **목표 직무/회사**: 시니어~리드급 프로덕트/UI 디자이너 채용 담당자, 또는 강연·컨설팅·디자인 교육 콘텐츠(책) 구매자. 단순 취업용이 아니라 "개인 브랜드/1인 비즈니스" 지향점이 강함.
- **경력 수준**: 약 20년 경력을 명시적으로 내세우는 것으로 보아 주니어보다는 시니어/리드 포지션 지원자 또는 이미 업계에서 인지도를 쌓은 프리랜서/1인 크리에이터로 보임.
- **효과성**: 채용 담당자에게는 "검증된 전문성 + 소통 가능한 인간미"를 동시에 전달하고, 잠재 고객/독자에게는 "책·디자인 시스템"이라는 구매 가능한 결과물을 자연스럽게 노출하는 이중 목적 페이지로서 효과적인 접근.
