# 나의 포트폴리오 기획서

참고 자료: `레퍼런스 분석 - adhamdannaway.md`, `레퍼런스 분석 - Behance 포트폴리오 템플릿 (Olivia Smith).md`, `컬러 팔레트 디자인 시스템.md`, `포트폴리오.png`

## 1. 나의 포지셔닝

- **목표 직무**: 풀스택 개발자
- **경력 수준**: 신입
- **지원 회사 성격**: 스타트업, 중소기업
- **나만의 강점**: 넓은 분야 대응력, 기술적 깊이, 호기심

**포지셔닝 한 줄 요약**
> "혼자서 기획 → 디자인 → 개발 → 배포까지 전체 사이클을 완주할 수 있는 신입 풀스택 개발자"

스타트업·중소기업은 신입에게 "여러 역할을 동시에 소화하는 능력"을 기대하는 경우가 많습니다. 지금 진행 중인 my-portfolio/novel-community/petlog_sns 세 프로젝트가 이미 기획서 작성 → UI 설계 → Supabase DB 설계(RLS 포함) → React 프론트엔드 → GitHub Actions 배포까지의 전 과정을 실제로 완주한 사례이므로, "말만 풀스택"이 아니라 "실제로 전체 사이클을 돌려본" 근거를 갖추고 있습니다.

## 2. Hero 섹션 컨셉

- **참고**: `포트폴리오.png`(Olivia Smith 템플릿)의 사진+헤드라인+CTA 2개+배지 구조. 다만 그 톤은 프리랜서 영업용이라 그대로 가져오지 않고, 톤만 "기술 중심형"으로 바꿔 적용합니다.
- **헤드라인 스타일**: 기술중심형 — "저는 OOO입니다" 대신 "무엇을 만들 수 있는가"를 코드/기술 키워드로 표현.
  - 예시 카피: `React와 Supabase로, 아이디어를 실제로 동작하는 서비스로 만듭니다.`
  - adhamdannaway.com의 `<coder>` 처럼 괄호·태그를 그대로 텍스트에 쓰는 기법을 응용해도 좋음 (예: `<FullstackDev />`).
- **시각적 요소**: 미니멀 + 애니메이션
  - 배경은 비우고, 코드 에디터/터미널 프레임 안에서 한 줄씩 타이핑되는 애니메이션 연출.
  - 또는 기술 스택 아이콘이 순차적으로 페이드인되는 미니멀한 모션.
- **색상 톤**: 다크 + 초록계열 네온 → `컬러 팔레트 디자인 시스템.md`(CHZZK 네온 그린 `#00FFA3`)를 그대로 적용. 이미 `theme.js`에 구현되어 있어 추가 작업 없이 재사용 가능.
- **차별화 포인트 (제안)**: 정적 문구 대신 "지금도 코드를 치고 있는 개발자"라는 인상을 주는 라이브형 연출 —
  - Hero 안에 `git commit -m "..."` 같은 실제 명령어가 타이핑되듯 흘러가는 애니메이션을 넣거나,
  - 최근 작업한 프로젝트명이 순환하며 노출되는 작은 위젯을 배치.
  - "소개하는 포트폴리오"가 아니라 "작업 중인 포트폴리오"라는 인상으로 차별화.

## 3. About Me 섹션 구성

- **참고**: `포트폴리오.png`의 "Who is Olivia Smith?" — 사진+한줄소개+통계 뱃지 구성이 인상적이었으나, 프리랜서향 통계(600+ 프로젝트 등)는 신입 포지셔닝과 맞지 않아 제외.
- **내 이야기 구성**
  - **개발 시작 스토리**: 고등학생 때 개발에 관심 → 대학생 때 적성을 발견하고 본격적으로 방향을 잡음. 타임라인(고등학생 → 대학생 → 지금) 형태로 시각화.
  - **핵심 가치관**: 논리적 사고, 문제 해결 능력 — 짧은 카피 + 아이콘 카드 2개로 표현.
  - **개인적 매력**: 볼링, 체스, 호기심, 새로운 기술
    - 제안: 볼링·체스는 둘 다 "수를 읽고 전략을 세우는" 활동이라는 공통점이 있어, "문제 해결 능력"이라는 가치관과 자연스럽게 연결해 한 문장으로 엮으면 개성이 더 뚜렷해짐.
      예: "체스와 볼링에서 다음 수를 계산하듯, 코드에서도 다음 단계를 미리 그려보는 걸 좋아합니다."
  - **성장 목표**: AI를 활용한 풀스택 개발자
- **구성 제안**: 좌측에 프로필 사진 + 한 줄 소개, 우측에 개발 시작 스토리 타임라인, 하단에 가치관 카드 2개 + "지금 관심있는 것"(호기심 · AI 목표)을 배치.

## 4. Skill 섹션 차별화

- **참고**: Olivia 템플릿의 원형 퍼센트 아이콘 대신, 요청하신 **프로그레스바** 방식 채택.
- **주력 기술 (5개)**: HTML 90% · CSS 80% · JavaScript 70% · Java 70% · Python 80%
- **표현 방식**: 가로 프로그레스바, 채움 색상은 브랜드 네온 그린. 스크롤 진입 시 0% → 목표치까지 차오르는 애니메이션(5번 항목의 "화려한 애니메이션" 요청과 연결).
- **차별화 제안**: 숫자(%)만으로는 근거가 약하므로, 각 스킬 바에 "이 기술로 만든 프로젝트" 배지를 연결 — 예를 들어 JavaScript 바 옆에 작은 태그로 "petlog_sns", "novel-community"를 붙이고 클릭하면 Projects 탭의 해당 카드로 스크롤 이동. Skill 섹션과 Projects 섹션을 서로 근거로 연결하는 구조.

## 5. 전체 디자인 방향

- **컬러 팔레트**: `컬러 팔레트 디자인 시스템.md` 그대로 사용 (이미 `theme.js`에 구현 완료 — 추가 작업 불필요).
- **폰트 스타일**: 개성있는 — 본문은 현재 Roboto 유지(가독성), 헤드라인/로고 등 포인트 영역에만 개성 있는 폰트(예: 모노스페이스 계열로 "코더" 정체성 강화)를 부분 적용.
- **레이아웃 스타일**: 매거진 — 현재 `SectionBand`의 중앙정렬 단일 컬럼 구조에서 한 단계 발전시켜, 섹션마다 큰 이미지/숫자 블록과 텍스트 블록을 좌우로 교차 배치하는 비대칭 그리드를 적용. (예: About은 이미지 왼쪽·텍스트 오른쪽, Skill은 반대로 텍스트 왼쪽·그래프 오른쪽)
- **애니메이션 수준**: 화려함 — 스크롤 리빌(fade+slide), Hero 타이핑 애니메이션, 프로젝트 카드 hover 네온 글로우(Projects 탭에 이미 적용된 효과를 전체로 확장), Skill 프로그레스바 채움 애니메이션.
- **주의사항**: adhamdannaway.com 분석에서 확인했듯, 스크롤 트리거 애니메이션에만 콘텐츠를 의존시키면 초기 렌더링/접근성 측면에서 약점이 될 수 있음. 화려한 연출은 유지하되, 콘텐츠 자체는 애니메이션 여부와 무관하게 항상 노출되도록(애니메이션은 등장 연출일 뿐, 최종 상태는 항상 보이도록) 구현할 것.

## 6. 차별화 전략 (신규 제안)

- **경쟁자 대비 내 강점**: 대부분의 신입 포트폴리오는 "학습한 결과물"을 캡처 이미지로 나열하는 데 그칩니다. 저는 기획서 → UI/UX 설계 → DB 스키마(RLS 포함) → 프론트엔드 → CI/CD 자동 배포까지, **한 사람이 전체 사이클을 실제 커밋과 문서로 남긴** 과정을 그대로 보여줄 수 있습니다. Live Demo가 실제로 동작하는 Supabase 백엔드와 연결되어 있어 "그럴듯해 보이는 화면"이 아니라 "지금 눌러볼 수 있는 서비스"라는 점이 가장 큰 차별점입니다.
- **타겟 어필 포인트**: 스타트업·중소기업은 신입에게 "혼자서도 굴러가게 만드는 능력"을 기대합니다. Projects 탭에서 각 프로젝트의 Live Demo를 직접 눌러 로그인·글쓰기·좋아요 같은 실제 CRUD 동작을 채용 담당자가 스스로 체험하게 만드는 것 자체가, 정적 스크린샷보다 훨씬 강한 설득력을 갖습니다.
- **독창적 아이디어**: "이 포트폴리오 사이트 자체를 하나의 프로젝트 사례"로 노출하는 메타 구조를 제안합니다. About Me의 성장 목표("AI를 활용한 풀스택 개발자")와 자연스럽게 연결해, "이 포트폴리오는 Claude Code와 함께 기획 → DB 설계 → 트러블슈팅 → 배포까지 진행했습니다"라는 짧은 데브로그 섹션(또는 블로그 글 1편)을 추가하는 것입니다. 예를 들어 실제로 겪었던 "썸네일 API(thum.io)가 SPA 로딩 화면을 캡처하는 문제를 옵션 조합으로 해결한 과정" 같은 실전 트러블슈팅 스토리를 짧게 공유하면, 코드만으로는 드러나지 않는 **문제 해결 과정 자체**를 직접 증명할 수 있어 "AI를 도구로 능숙하게 다루는 개발자"라는 포지셔닝을 시각적으로 뒷받침합니다.
