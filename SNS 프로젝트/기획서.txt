=====================================
모바일 서비스 UI 기획서 — PetLog
=====================================

1. 서비스 기본 정보
-------------------------------------
서비스 이름: PetLog (펫로그)

서비스 주제:
  반려동물 라이프스타일 SNS
  강아지·고양이·소동물 등 반려동물과의 일상을
  짧은 피드 형식으로 공유하고, 비슷한 취향의
  펫 오너들과 소통하는 감성 커뮤니티

대상 플랫폼:
  1차: 모바일 웹 (반응형, PWA 설치 지원)
  향후: 네이티브 앱(iOS·Android) 전환 검토
  ※ AR 필터·GPS 트래킹·앱 아이콘 뱃지 등 네이티브 의존
    기능은 앱 전환 시점에 구현하며, 웹에서는 대체 UI
    (탭바 뱃지 등)로 제공한다.

핵심 가치:
  1. 반려동물 중심 피드
     - 사람이 아닌 반려동물이 프로필의 주인공
     - 펫 프로필 단위로 각자의 성격과 일상을 기록
  2. 감성 필터 & 스티커
     - 반려동물 전용 AR 필터와 귀여운 스티커 팩 제공
     - 사진 한 장도 특별하게 남기는 편집 경험
  3. 건강 기록 연동
     - 일상 피드와 함께 예방접종·건강검진 기록을 타임라인에 통합
     - SNS와 건강 관리를 한 곳에서 해결

계정 구조 (핵심 개념):
  - 사람(보호자) 계정 1개 아래에 펫 프로필을 N개 등록하는 구조
  - 보호자는 회원가입 시 닉네임을 등록 (계정 설정·차단 목록 등
    보호자 단위 화면에 표시, 피드의 주인공은 여전히 펫 프로필)
  - 로그인·설정·알림은 보호자 계정 단위로 관리
  - 게시물 작성·피드 노출·팔로우는 "펫 프로필" 단위
    (예: 같은 보호자의 뭉치는 팔로우하고 나비는 안 할 수 있음)
  - 좋아요·댓글의 주체는 현재 선택된 펫 프로필로 표시

타겟 사용자:
  주 타겟: 반려동물을 키우는 15~35세
  ※ 만 14세 미만은 가입 불가 (개인정보보호법상 법정대리인
    동의 절차 회피 — 회원가입 시 생년월일로 확인)

  PRIMARY   — SNS에 익숙하고 반려동물 사진 공유에 적극적인
              15~24세 펫 오너 (강아지·고양이 위주)
  SECONDARY — 반려동물을 가족처럼 여기며 건강 정보와
              커뮤니티를 함께 찾는 25~35세 직장인
  TERTIARY  — 입양 예정자, 반려동물 관련 소상공인
              (펫샵·동물병원·수제 간식 판매자)
              ※ 소상공인 대상 기능은 "9. 수익 모델 로드맵" 참고

색상 사용 원칙:
  - 본문·주요 텍스트는 딥 그린 또는 차콜 계열만 사용
  - 웜 샌드(#E7E1B1)는 장식용 테두리·보조 배경 한정
    (크림 배경과 대비가 낮으므로 텍스트·아이콘에는 사용 금지)
  - 텍스트 대비율은 WCAG AA(4.5:1) 이상,
    아이콘·UI 컴포넌트는 3:1 이상을 유지

폰트:
  - 로고·영문 타이틀: Syne ExtraBold (Google Fonts)
    → 강한 개성과 친근함이 공존하는 디스플레이 폰트
    ※ Syne는 라틴 전용 폰트이므로 한글에는 적용되지 않음
  - 한글 제목: Noto Sans KR Black (900)
    → 별도 폰트 추가 없이 제목 위계 표현
  - 본문: Noto Sans KR Regular/Medium (Google Fonts)
    → 어떤 크기에서도 가독성이 좋은 본문 폰트

전체 분위기:
  "풀밭 위의 햇살처럼 싱그럽고 따뜻한"
  자연에서 온 그린과 크림의 조화로 반려동물과 함께하는
  야외 산책의 싱그러움을 표현.
  과하게 귀엽지 않으면서도 자연 친화적이고 건강한 감성.

  키워드: 따뜻한 감성 / 자연 친화적 / 귀엽고 발랄

참고 사이트:
  - Instagram  → 피드 레이아웃 · 스토리 구조 참고
  - BeReal     → 일상 기록 · 진정성 피드 참고

3. 필요한 페이지 목록
-------------------------------------
[✓] 1. 로그인 · 회원가입 페이지 (Onboarding)
    - 기능:
      · 이메일 + 비밀번호 로그인
      · 카카오 · 구글 소셜 로그인 (원탭) — 2차 개발에서 추가
      · 이메일 회원가입 + 이메일 인증
      · 닉네임(보호자 이름) 입력 — 중복 확인 후 저장
      · 생년월일 입력 → 만 14세 미만 가입 차단
      · 비밀번호 찾기 (이메일 발송)
      · 자동 로그인 (토큰 저장)
    - 특징:
      · 회원가입 완료 후 반려동물 프로필 온보딩 화면으로 자동 이동
      · 소셜 로그인 도입 후에는 소셜 버튼을 이메일보다 상단 배치 — 마찰 최소화
      · 로그인 성공 시 메인 피드로 fadeIn 전환 애니메이션
      · Floating Label 방식 입력 필드

[✓] 2. 메인 피드 (홈 화면) — Home Feed
    - 기능:
      · 팔로잉 기반 피드 — 최신순 정렬
      · 스토리 영역 (상단 원형 아바타 행) — 3차 개발, 1차에서는 미노출
      · 무한 스크롤 (Infinite Scroll)
      · 게시물 좋아요 · 댓글 · 공유 · 저장
      · Pull to Refresh (당겨서 새로고침)
      · 탐색 탭 전환 (팔로잉 / 추천)
    - 특징:
      · 카드형 피드 — 사진 상단, 반려동물 이름·품종 태그 하단 표시
      · 이중 탭(더블탭)으로 좋아요 — 하트 애니메이션 표시
      · 스크롤 방향에 따라 하단 탭바 자동 숨김/표시
      · "추천" 탭은 유사 반려동물 종류 기반 알고리즘 피드
        (알고리즘은 3차 개발 — 그 전까지는 전체 최신 게시물 표시)

[✓] 3. 탐색 · 검색 페이지 — Explore
    - 기능:
      · 해시태그 · 반려동물 이름 검색
      · 인기 게시물 그리드 (3열)
      · 카테고리 필터 (강아지 / 고양이 / 소동물 / 기타)
      · 인기 해시태그 추천 목록
      · 팔로우할 만한 펫 프로필 추천
    - 특징:
      · 검색창 포커스 시 최근 검색어 드롭다운 표시
      · 그리드 첫 번째 칸은 2배 크기 — 인기 게시물 강조
      · 스크롤 시 검색창이 상단에 sticky 고정
      · 1차 개발에서는 검색 없이 "전체 최신 게시물 그리드"만 표시,
        검색·필터·추천은 2차에서 완성

[✓] 4. 게시물 상세 페이지 — Post Detail
    - 기능:
      · 이미지 슬라이더 (다중 사진 → 좌우 스와이프)
      · 좋아요 수 · 댓글 목록 표시
      · 댓글 작성 + 이모지 반응 + 답글(대댓글, 1단계까지)
      · 해시태그 클릭 → 탐색 페이지 이동
      · 위치 태그 표시 (동 단위까지만 — 7. 정책 참고)
      · 더보기 메뉴

        - 타인 게시물: 신고 / 차단 / 공유 / 저장
      · 공유 = 게시물 링크 복사 + OS 공유 시트 호출
    - 특징:
      · 이미지 하단에 점(dot) 인디케이터로 장수 표시
      · 댓글 입력창은 하단에 고정 — 키보드 올라올 시 자동 올라옴
      · 피드에서 상세 페이지 진입 시 공유 Element 트랜지션
      · 본문 3줄 초과 시 "더 보기" 버튼으로 축약 표시

[✓] 5. 게시물 작성 페이지 — Create Post

      · 카메라 직접 촬영
      · 반려동물 전용 AR 필터 적용 (3차 개발)
      · 텍스트 본문 작성 (최대 500자)
      · 해시태그 자동 완성 (#강아지 입력 시 추천)
      · 위치 태그 추가 (동 단위 선택)
      · 어느 펫 프로필 피드에 올릴지 선택
    - 이미지 업로드 스펙:
      · 포맷: JPEG · PNG · WebP
      · 원본 최대 10MB/장, 업로드 시 긴 변 1440px로 리사이즈
      · 비율: 1:1 기본, 4:5 ~ 1.91:1 범위 내 크롭 허용
    - 특징:
      · 이미지 선택 → 편집 → 본문 작성 3단계 흐름
      · 임시저장 기능 — 작성 중 나가면 "임시저장하시겠습니까?" 팝업
      · 하단 탭바 중앙 FAB 버튼으로 진입 — 항상 접근 가능
      · 해시태그 입력 시 #뒤 단어를 자동으로 강조 표시

[✓] 6. 마이페이지 · 프로필 — My Profile

      · 팔로워 / 팔로잉 / 게시물 수 통계 (펫 프로필 단위)

      · 건강 기록 탭 (접종·검진 히스토리)

      · 프로필 편집 · 설정 (차단 목록 관리 포함)
    - 특징:

      · D-Day 표시 — "뭉치의 생일이 14일 남았어요 🎂"
      · 스크롤 시 프로필 헤더가 접혀서 상단에 미니 프로필로 고정
      · 건강 기록은 타임라인 형식으로 날짜순 정렬

[✓] 7. 알림 페이지 — Notifications
    - 페이지명: 알림

      · 읽음 / 안 읽음 구분 표시
      · 일괄 읽음 처리
    - 특징:
      · 알림 항목 좌우 스와이프로 삭제

      · 미확인 알림 수는 하단 탭바 알림 아이콘에 뱃지로 표시
        (네이티브 앱 전환 시 앱 아이콘 뱃지로 확장)
      · 1차 개발에서는 빈 상태 화면("아직 알림이 없어요")만 표시,
        알림 수집·표시는 2차에서 완성


4. 핵심 기능 정의
-------------------------------------
필수 기능 (꼭 있어야 하는 기능)
1. 펫 프로필 생성 · 관리
   이름·품종·생년월일·성별·사진을 등록하는 반려동물 중심 프로필.
   보호자 계정 1개당 복수 펫 프로필 추가 지원.

2. 사진 게시물 작성 · 수정 · 삭제
   최대 10장의 사진과 텍스트, 해시태그, 위치 태그를 포함한
   게시물 CRUD 기능. 수정·삭제는 상세 페이지 더보기 메뉴에서 진입.

3. 팔로우 / 언팔로우 시스템
   펫 프로필 간 비대칭 팔로우.
   팔로잉한 펫 프로필의 게시물이 홈 피드에 표시됨.

4. 좋아요 · 댓글 인터랙션
   게시물 좋아요(더블탭 포함)와 댓글 작성.
   댓글에 이모지 반응, 답글(대댓글) 1단계 지원.

5. 이메일 로그인 + 소셜 로그인 (카카오 · 구글)
   1차는 이메일 회원가입·로그인으로 시작하고,
   2차에서 OAuth 2.0 기반 카카오·구글 원탭 로그인을 추가.
   (소셜 로그인은 플랫폼 키 발급·검수 리드타임 고려)

부가 기능 (있으면 좋은 기능)
1. 건강 기록 타임라인
   예방접종·건강검진·구충 날짜를 기록하고
   D-day 리마인더 알림을 보내는 헬스 로그 기능.

2. 스토리
   24시간 후 자동 만료되는 세로형 사진 스토리.
   홈 피드 상단 원형 아바타 행에서 진입, 열람 여부 링 표시.

3. 반려동물 전용 AR 필터
   귀·꼬리·왕관 등 동물 AR 스티커를 실시간 카메라에 합성.
   게시물 작성 시 적용 가능. (네이티브 앱 전환 후 구현)

4. 1:1 다이렉트 메시지
   팔로잉한 사용자와의 개인 채팅.
   사진 공유 및 이모지 반응 지원.

5. 게시물 저장 · 컬렉션
   마음에 드는 게시물을 저장하고 주제별 컬렉션으로 분류.
   마이페이지에서 확인 가능.

6. 산책 기록 (GPS 트래킹)
   산책 경로를 지도에 기록하고 거리·시간·소비 칼로리를 표시.
   게시물에 첨부 가능. (경로 공개 범위는 7. 정책 참고)


5. 모바일 UI 특징
-------------------------------------
네비게이션:
  하단 탭바 방식 채택.
  엄지손가락이 닿기 쉬운 화면 하단에 4개 탭 배치.
  중앙에 게시물 작성 FAB 버튼으로 핵심 행동 유도.

  탭 구성:
    [홈] [탐색] [+ 작성(FAB)] [알림] [마이]

주요 버튼 위치:
  - FAB (게시물 작성): 하단 탭바 중앙 — 항상 접근 가능
  - 좋아요 버튼: 게시물 카드 하단 좌측
  - 댓글 작성창: 상세 페이지 하단 고정

터치 영역 크기:
  모든 터치 가능한 요소 최소 44×44px (iOS HIG 기준)
  - 하단 탭 각 항목 높이: 56px
  - FAB 버튼 지름: 56px
  - 댓글 좋아요 버튼: 44×44px
  - 버튼 간 최소 간격: 8px 이상

제스처 인터랙션:
  - 더블탭       → 좋아요 (하트 애니메이션)
  - 좌우 스와이프 → 이미지 슬라이더
  - 위 스와이프   → 댓글 시트 열기
  - 길게 누르기   → 게시물 옵션 메뉴
  - 당겨서 새로고침 → 피드 갱신

세로/가로 모드:
  기본: 세로 모드(Portrait)만 지원
  예외: 사진 편집 화면, AR 필터 적용 화면은
       가로 모드(Landscape) 전환 허용

접근성:
  - 시스템 다크모드 자동 감지 및 대응 (2. 디자인 컨셉의
    다크 모드 색상 팔레트 적용)
  - 텍스트 대비율 WCAG AA 기준 (4.5:1) 이상 유지
  - 시스템 폰트 크기 설정에 반응 (rem 기반 타이포그래피)


6. 개발 우선순위
-------------------------------------
원칙:
  핵심 가치(반려동물 기록 + 공유)를 경험할 수 있는
  최소 기능 세트(MVP)를 1차에 완성한다.
  이후 피드백을 받으며 2·3차를 순차 추가한다.

1차 개발 (MVP 핵심) — 가장 먼저 만들 페이지와 기능
  목표: "사진을 올리고 팔로워와 공유하는" 핵심 경험 완성
  - 로그인 · 회원가입 (이메일 로그인 + 펫 프로필 등록 온보딩)
  - 메인 피드 (팔로잉 피드 + 좋아요 · 댓글,
    추천 탭은 전체 최신 게시물로 임시 운영)
  - 게시물 작성 · 수정 · 삭제 (사진 업로드 + 텍스트 + 해시태그)
  - 마이페이지 기본 (내 게시물 그리드 + 팔로우 통계)
  ※ 탐색 탭: 전체 최신 게시물 그리드만 표시
  ※ 알림 탭: 빈 상태 화면으로 노출 (기능은 2차)
  ※ 스토리 영역: 미노출 (3차)

2차 개발 (참여 강화) — 그 다음 추가할 기능
  목표: 재방문율 높이기 — 건강 기록과 알림으로 앱 의존도 형성
  - 소셜 로그인 (카카오 · 구글 OAuth)
  - 탐색 · 검색 (해시태그·펫 프로필 검색 + 인기 피드 + 카테고리 필터)
  - 알림 시스템 (좋아요·댓글·팔로우 실시간 알림 + 탭바 뱃지)
  - 건강 기록 (접종·검진 타임라인 + D-day 알림)
  - 게시물 저장 (컬렉션 저장 + 마이페이지 탭)
  - 신고 · 차단 처리 프로세스 (7. 정책 참고)

3차 개발 (경험 고도화) — 여유가 있을 때 추가
  목표: 타 서비스와 완전한 차별화 — PetLog만의 고유 경험 완성
  - 스토리 (24시간 만료 세로형 스토리)
  - AR 필터 (반려동물 전용 실시간 AR 카메라 — 네이티브 앱 전환 후)
  - DM 다이렉트 메시지 (1:1 채팅 + 사진 공유)
  - 산책 GPS 트래킹 (경로 기록 + 칼로리 계산 — 네이티브 앱 전환 후)
  - 추천 알고리즘 (유사 반려동물 기반 피드 개인화)


7. 정책 및 안전
-------------------------------------
신고 · 차단:
  - 게시물·댓글 신고 사유: 스팸 / 동물 학대 의심 / 욕설·혐오 /
    부적절한 콘텐츠 / 기타
  - 신고 접수 시 관리자 검토 → 게시물 숨김·삭제, 반복 위반 시
    계정 이용 제한 (경고 → 7일 정지 → 영구 정지)
  - 동물 학대 의심 신고는 최우선 검토 대상으로 분류
  - 사용자 차단: 차단한 상대의 게시물·댓글·검색 노출 차단,
    차단 목록은 설정에서 관리

위치 정보 보호:
  - 위치 태그는 "동 단위"까지만 선택·표시 (상세 주소·좌표 비공개)
  - 산책 GPS 경로는 기본 비공개, 공유 시 출발·도착 지점
    반경 200m 마스킹 처리

개인정보:
  - 만 14세 미만 가입 불가 (회원가입 시 생년월일 확인)
  - 회원 탈퇴 시 게시물·댓글·건강 기록 즉시 삭제
    (법정 보관 의무 항목 제외)

커뮤니티 가이드라인:
  - 동물 학대·유기 조장 콘텐츠 금지
  - 무허가 분양·판매 게시물 금지
  - 타인(타 반려동물) 비방·혐오 표현 금지


8. 성공 지표 (KPI)
-------------------------------------
1차 (MVP) 검증 지표:
  - 가입 완료율: 가입 시작 대비 펫 프로필 등록 완료 60% 이상
  - 첫 게시물 작성률: 가입자의 40% 이상이 7일 내 첫 게시물 작성

2차 (참여 강화) 지표:
  - 주간 리텐션(W1): 30% 이상
  - DAU/MAU 비율: 25% 이상
  - 건강 기록 사용률: 활성 사용자의 20% 이상이 기록 1건 이상 등록

3차 (고도화) 지표:
  - 팔로우 그래프 밀도: 활성 사용자당 평균 팔로잉 10개 이상
  - 추천 피드 체류 시간: 팔로잉 피드 대비 80% 수준 도달


9. 수익 모델 로드맵 (장기)
-------------------------------------
TERTIARY 타겟(펫샵·동물병원·수제 간식 판매자)을 위한
비즈니스 기능은 사용자 기반 확보 후 도입한다.

  - 비즈니스 프로필: 업체 인증 뱃지, 영업시간·예약 링크 표시
  - 지역 기반 노출: 탐색 페이지 내 "우리 동네 펫 플레이스" 섹션
  - 제휴 콘텐츠: 동물병원 연계 건강 정보 콘텐츠, 접종 리마인더
    연동 예약
  - 광고: 피드 네이티브 광고 (명확한 광고 표기, 빈도 제한)
